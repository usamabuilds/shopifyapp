# Phase 1 Internal Launch Readiness Checklist

> Internal checklist for listing/review/beta onboarding readiness (no new feature work).

## A) Product truth and scope

- [ ] Listing copy only claims currently shipped app foundations.
- [ ] No mention of V2 capabilities (shared inbox/chatbot/CRM/non-WhatsApp channels).
- [ ] Consent limitation is explicitly documented where relevant.

## B) Listing collateral readiness

- [ ] One-line and full description drafts prepared.
- [ ] Key feature bullets align to existing navigation and behavior.
- [ ] Reviewer-safe claims checklist completed.

## C) Reviewer/tester package readiness

- [ ] Reviewer quick-flow steps documented.
- [ ] Guardrail/blocking states documented.
- [ ] Empty-state expectations documented.

## D) Merchant onboarding/help readiness

- [ ] First-run guide maps directly to current UI sections.
- [ ] Safety guidance included (utility vs marketing eligibility responsibility).
- [ ] Links/pointers to support tools included.

## E) FAQ/support readiness

- [ ] FAQ answers top blocking issues seen in current forms.
- [ ] Troubleshooting quick checks are actionable from existing UI.
- [ ] Escalation payload requirements are documented.

## F) Internal operations readiness

- [ ] Team can use support tools tables for triage and recovery.
- [ ] Team has runbook references for deeper incidents.
- [ ] Team understands support actions are recovery-only for existing due work.

## G) Documentation wiring

- [ ] Admin app README points to this launch collateral pack.
- [ ] Existing hardening/runbook docs remain linked for operations.
