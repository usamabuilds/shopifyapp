# Phase 1 Shopify App Listing Copy Draft (Foundation-Accurate)

> Purpose: draft listing copy that reflects the current Phase 1 app behavior only.

## 1) App name options (working)

- **WhatsApp Commerce Foundations**
- **WhatsApp Ops for Shopify (Phase 1)**

## 2) One-line value proposition

Run WhatsApp message foundations for Shopify order updates, cart recovery, and broadcast campaigns from one embedded admin app.

## 3) Short app description (for listing)

This embedded Shopify app helps merchants set up and operate practical WhatsApp messaging foundations: order confirmation, order status updates, abandoned cart recovery, and broadcast campaigns. Teams can configure template mappings, manage automation toggles, monitor operational state, and use built-in support tooling for basic recovery actions.

## 4) Full app description (honest Phase 1 version)

This app is focused on launch-safe operations for a Phase 1 WhatsApp commerce foundation.

What merchants can do today:

- Configure merchant profile defaults and onboarding readiness in **Settings**.
- Review template-to-flow mappings in **Templates** with category checks for utility vs marketing use cases.
- Enable/disable **Order confirmation**, **Order status updates**, and **Abandoned cart recovery** in **Automations**.
- Create and run **Broadcast campaigns** (draft, schedule, send-now) in **Campaigns**.
- Create, validate, and publish simple **Workflows** using supported blocks (trigger, delay, condition, send message, end).
- Review aggregate activity and outcome summaries in **Analytics**.
- Use **Support tools** for webhook/outbound visibility, failure triage, and limited recovery actions.

Current Phase 1 boundaries to keep explicit in listing copy:

- This release is a foundations release; it does not include advanced CRM/inbox/chatbot capabilities.
- Consent handling is not enforced per contact by automation logic yet; merchants/operators remain responsible for message eligibility.
- Messaging outcomes depend on valid template keys and proper template category alignment.

## 5) Key features bullets (listing-friendly)

- Embedded Shopify admin experience with role-friendly navigation.
- Template mapping checks for order, cart recovery, and campaign use cases.
- Automation controls for order confirmation, status updates, and cart recovery.
- Broadcast campaign drafting, scheduling, send-now queueing, and logs.
- Workflow builder foundation with draft/publish/pause/archive transitions.
- Support tooling for operational alerts, traceability, and guided recovery actions.

## 6) Suggested “How it works” bullets

1. Install app and open embedded admin.
2. Complete settings and readiness basics.
3. Confirm template mappings for each enabled flow.
4. Enable automations and/or queue campaigns.
5. Monitor analytics and support tools for outcome status.

## 7) Suggested permissions/scopes explainer text

The app requires Shopify access needed to authenticate the embedded admin experience and process order/checkout/fulfillment events that drive configured WhatsApp foundations.

## 8) Support/contact listing snippet

Support teams should direct merchants to in-app **Support tools** first, then provide assisted triage using webhook intake state, outbound message state, and campaign/workflow operational panels.
