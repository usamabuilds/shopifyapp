# PRODUCT_BRIEF.md

## Product

Shopify WhatsApp App

## Product one-liner

WhatsApp commerce automation for Shopify merchants.

## Positioning

A Shopify-first WhatsApp automation app focused on practical commerce flows:
- order confirmation
- post-purchase updates
- abandoned cart recovery
- promotional broadcasts

## Anti-positioning

This is not a full CRM, not a chatbot platform, and not a general multi-channel customer engagement suite in Phase 1.

## Core value proposition

Help merchants automate high-value WhatsApp communication from Shopify data with clear setup, clear control, and measurable outcomes.

## Main merchant problems

1. order communication is manual and inconsistent
2. post-purchase updates take time and create support load
3. abandoned carts are not followed up systematically
4. promotional outreach is fragmented
5. there is no simple operational view of what was sent and what worked

## Ideal customer profile

### Best fit
- Shopify merchants
- WhatsApp is already important in their market or workflow
- need order and customer messaging automation
- want practical automation, not enterprise complexity
- comfortable installing a Shopify app and mapping templates

### Less ideal
- merchants who want only a live support inbox
- merchants needing full CRM
- merchants needing heavy custom telephony logic
- merchants needing many channels in V1

## Primary user roles

### Buyer
Founder, operator, or growth lead.

### Daily operator
Ops person, store manager, or marketing/operator hybrid.

## Main use cases

### 1. Order confirmation
When a new order is placed, send a structured WhatsApp confirmation automatically.

### 2. Order status updates
When order status changes, send mapped updates like shipped or delivered.

### 3. Abandoned cart recovery
When a checkout is left incomplete, send a recovery message after a configured delay.

### 4. Broadcast campaigns
Let merchants send controlled promotional campaigns to an eligible audience.

### 5. Basic automation control
Let merchants configure simple trigger/action logic without needing a broad platform.

## Why merchants would adopt

- saves operator time
- improves consistency
- reduces manual follow-up
- increases recovery opportunities
- gives visibility into message outcomes

## Alternatives merchants use now

- manual WhatsApp sending
- fragmented operational workflows
- generic automation tools
- disconnected campaign tools
- agency-led manual setup

## Why this product can win

- Shopify-native
- focused scope
- operational rather than bloated
- clear setup path
- direct business-use cases
- measurable messaging outcomes

## Product success in first 14 days

A successful merchant should:
- install the app
- complete onboarding
- connect WhatsApp
- sync templates
- enable at least one automation
- send first successful message
- see message status in the app

## Phase 1 success signal

If merchants can go from install to first live automation and clearly understand results, Phase 1 is doing its job.
